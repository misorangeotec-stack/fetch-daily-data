# Sales Person Mapping — Ledger Name → Sales Person

The fetch tool (`tools/fetch_tally_credit_limits.py`) uses this mapping to
populate the **Sales Person** column on each ledger row pulled from Tally.

The mapping was seeded from `references/Credit Limit & Opening.xlsx` and is
keyed by ledger name (`$Name`) only — the same customer in Noida vs. Surat
or in Enterprise vs. O-tec books gets the same sales person. To re-assign a
customer, edit the row in place; to onboard a new customer, append a row.

Unmapped ledger names emit a one-time stderr warning and the Sales Person
column is left blank — the script does not crash. Add a row to fix.

The first table (`## Sales Persons`) is the one parsed. Header row must be
`name | sales_person` (case-insensitive).
